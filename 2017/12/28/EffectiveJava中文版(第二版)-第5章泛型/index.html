Effective Java 中文版（第二版）第5章 泛型

第5章 泛型

5.1 请不要在新代码中使用原生态类型

  • Set
  • Set<?>:通配符类型，表示某种未知类型的集合
  • Set：原生态类型

前两种是安全的，后面一种是不安全的，不推荐使用。

5.2 消除非受检警告

可以采用@SuppressWarning警告，其中其注解范围越小越好。同时要把原因详细注释下来。

5.3 列表优于数组

数组和泛型列表的区别：

  • 数组是协变的（covariant），泛型是不可变的(invariant);
  • 数组是具体化的，在运行时才检查元素类型约束，泛型是通过擦除(erasure)来实现的，擦除就是使泛型可以与没有使用泛型的代码随意进行互用；

5.4 优先使用泛型

5.7 优先考虑类型安全的异构容器

如果需要以类型安全的方式访问数据库行的任意多的列，需要将键(key)进行初始化而不是将容器(container)参数化。然后将参数化的键提交给容器，来插入或者获取值。用泛型系统来确保值的类型和它的键相符。

Favorites的实现有两种局限性：

  • 值类型只能是Object；
  • 不能用在具体化的类型中，比如List等
